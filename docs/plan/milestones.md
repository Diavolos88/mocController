# Milestones

Опиши крупные этапы реализации. Каждый этап должен иметь понятный критерий готовности.

| Этап | Цель | Ключевые задачи |
| --- | --- | --- |
| Этап 1 | Реализовать базовый REST-сервис, который принимает конфиги в json и отдаёт их обратно | Есть рабочие эндпоинты для приёма/выдачи, заглушка может сохранить свой конфиг |
| Этап 2 | Хранить стартовый конфиг каждой заглушки и давать UI для сравнения | В UI рядом с каждым параметром отображается стартовое значение, есть кнопка «вернуть параметр» и «вернуть весь конфиг» |
| Этап 3 | Добавить локальное (на диске) хранение актуальных конфигов и возможность редактировать их в UI | Правки через интерфейс сохраняются, заглушки получают обновления без рестартов |
| Этап 4 | Реализовать API проверки обновления (версия конфига + имя заглушки) | Сервис по запросу говорит, нужно ли обновиться, и направляет на эндпоинт получения конфигурации |
| Этап 5 | Сделать поддержку отложенных изменений и управления задачами | Можно создать задачу с датой/временем (МСК), увидеть её статус и отменить до применения |

Далее будем дорабатывать по ходу
